# Positioning Checks

Use this reference to build related-work arguments.

## Source Patterns

- UNC Writing Center guidance on literature reviews emphasizes synthesis, patterns, debates, and gaps rather than source-by-source summary: https://writingcenter.unc.edu/tips-and-tools/literature-reviews/
- NeurIPS Paper Checklist asks authors to discuss assumptions, limitations, and broader context that affect claims: https://neurips.cc/public/guides/PaperChecklist
- Citation-integrity practice: verify that each cited paper supports the exact statement attached to it.

## Cluster Patterns

| Cluster type | Useful question |
| --- | --- |
| Method family | What algorithmic or architectural idea is shared? |
| Task setting | What problem, environment, robot, dataset, or modality is studied? |
| Benchmark | Which evaluation protocol defines comparability? |
| Data | What supervision, demonstrations, simulation assets, or sensors are used? |
| System | What deployment, latency, safety, or integration constraint matters? |
| Theory | What assumption or guarantee does prior work rely on? |

## Positioning Moves

- "Prior work on X establishes Y, but does not evaluate Z."
- "Methods in cluster A optimize B; our setting differs because C."
- "The closest work is P. It shares Q, while this paper differs in R."
- "We therefore narrow the claim to S rather than claiming general superiority."

## Red Flags

- Related work is a list of mini-summaries with no synthesis.
- Every paragraph ends with "however" but no concrete gap.
- A paper is cited only because it is famous.
- Prior work is framed as weak without naming the exact limitation.
- The current paper's contribution changes across abstract, introduction, and related work.
