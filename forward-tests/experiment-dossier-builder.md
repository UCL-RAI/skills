# Forward Test: Experiment Dossier Builder

## Skill

`experiment-dossier-builder`

## Prompt

Use `experiment-dossier-builder` to prepare a remote-to-local paper handoff
from this project snapshot.

```markdown
Repository state:
- commit: a13f9c2, branch exp-v2, dirty: configs/eval.yaml modified
- method code: src/model.py, src/policy.py
- evaluation: scripts/evaluate.py, scripts/make_table.py

Experiment artifacts:
- configs/base.yaml and runs/base_seed{1,2,3}.json
- configs/ours.yaml and runs/ours_seed{1,2,3}.json
- configs/no_memory.yaml and runs/no_memory_seed1.json
- figures/success_by_task.pdf generated by scripts/plot_success.py
- notes.md says "ours improves success by 12%"

Known gaps:
- no exact command history
- no environment lockfile
- seed 2 for ours contains two interrupted episodes
- the no-memory ablation has only one seed
```

## Expected Artifact Checklist

- Records the dirty repository snapshot and inspection scope.
- Inventories all experiment families before selecting headline results.
- Gives each experiment, result, claim, and source a stable id.
- Treats the 12% note as unverified until it is traced to result artifacts and
  aggregation code.
- Records interrupted episodes and one-seed ablation as evidence limitations.
- Includes method map, result ledger, claim candidates, failures, figure
  inventory, provenance summary, evidence gaps, and source index.
- Uses repository-relative source locators and contains no credentials or
  machine-specific absolute paths.

## Pass Conditions

- Every paper-relevant result has a source locator and valid evidence status.
- The dossier distinguishes observed results from narrative interpretation.
- Missing commands, environment capture, and ablation coverage remain visible.
- The local writer could identify what is usable and what needs another run.

## Fail Conditions

- The response repeats the 12% claim as verified based only on `notes.md`.
- It hides the dirty config or interrupted episodes.
- It summarizes only the best run or invents missing seeds, commands, metrics,
  or environment details.
