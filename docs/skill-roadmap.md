# Skill Roadmap

This roadmap keeps the first public release small enough to review while covering the full Robotics & AI research loop.

## Release 0.1

| Skill | Type | Purpose |
| --- | --- | --- |
| `rai-research-flow` | flow | Route research onboarding, survey, project ideation, and paper-writing tasks. |
| `rai-paper-flow` | flow | Route conference-paper writing and audit workflows for Robotics & AI venues. |
| `rai-coding-flow` | flow | Route daily Robotics & AI code work through design, debugging, testing, review, and reproducibility checks. |
| `knowledge-base-onboarding` | atomic | Turn scattered papers, notes, code, and datasets into a source inventory, concept map, and next-action plan. |
| `paper-search-protocol` | atomic | Build reproducible search logs across papers, projects, datasets, and code. |
| `paper-reading-card` | atomic | Convert one paper into a structured, source-grounded reading card. |
| `evidence-matrix-builder` | atomic | Turn paper/project cards into comparison matrices and gap maps. |
| `research-plan-grill` | atomic | Resolve ambiguous research and engineering decisions one question at a time before execution. |
| `research-idea-rubric` | reference | Score and refine project ideas for novelty, feasibility, baselines, and evaluation. |
| `venue-paper-outline` | reference | Plan papers for ICRA, RSS, CoRL, ICML, ICLR, and NeurIPS-style venues. |
| `related-work-positioning` | atomic | Build source-grounded prior-work clusters, novelty boundaries, and positioning language. |
| `abstract-introduction-builder` | atomic | Draft abstracts and introductions from verified problem, gap, method, evidence, contribution, and limitation material. |
| `paper-draft-builder` | atomic | Turn verified experiment and literature artifacts into a source-grounded conference-paper story and draft. |
| `survey-synthesis-builder` | atomic | Build a taxonomy, claim-evidence structure, section contracts, and survey draft from verified literature. |
| `citation-integrity-auditor` | atomic | Check whether citations are real and actually support each claim. |
| `manuscript-structure-auditor` | atomic | Audit coherence, section flow, paragraph logic, repetition, contradictions, and claim threading. |
| `scientific-writing-editor` | atomic | Edit grammar, clarity, concision, scientific tone, and generic AI-style prose patterns without changing technical claims. |
| `benchmark-audit` | atomic | Validate benchmark fit, metrics, baselines, ablations, statistical reporting, leakage risk, and claim-result alignment. |
| `experiment-provenance-auditor` | atomic | Check whether paper results trace back to configs, code versions, data versions, seeds, logs, hardware, and plots. |
| `experiment-dossier-builder` | atomic | Summarize a remote project and its results into a portable, source-grounded Markdown handoff. |
| `paper-code-consistency-auditor` | atomic | Audit whether manuscript descriptions, configs, evaluation, and generated artifacts match the implementation. |
| `limitations-failure-case-auditor` | atomic | Audit limitations, assumptions, failure cases, negative results, and deployment or ethics risks. |
| `scientific-figure-director` | tool | Plan, prompt, and audit scientific figures and diagrams. |
| `paper-red-team-review` | atomic | Perform harsh but evidence-grounded paper review across facts, logic, writing, and visuals. |
| `reviewer-response-builder` | atomic | Build point-by-point reviewer response plans and revision actions. |
| `latex-submission-checker` | tool | Check LaTeX source packages for buildability, venue compliance, anonymity, arXiv, and submission hygiene. |
| `slide-talk-builder` | tool | Build timed research talks with a rendered deck, speaker notes, and visual QA. |
| `robotics-ai-coding-flow` | atomic | Apply Robotics & AI code quality checks around ROS, simulation, ML experiments, and reproducibility. |

## Release 0.2 Candidates

- `artifact-release-packager`: prepare public code, data, model, and artifact-release bundles.
- `paper-to-presentation-flow`: route paper material into talks, posters, and demos.

## Non-Goals

- Do not create a single mega-skill that claims to do all research.
- Do not bundle third-party repositories.
- Do not encode venue deadlines or time-sensitive rules in skill bodies.
- Do not make paper-writing skills fabricate citations, benchmarks, or claims.
